# 日语体

核心特征：**敬语分层** + **主语省略** + **含蓄表达** + **固定客套构造**。日语风格切换主要通过**语尾**和**敬语等级**完成。

## 敬语三层

| 等级 | 名称 | 用途 | 例 |
|---|---|---|---|
| 丁寧語 | です / ます | 对陌生人、正式场合 | 行きます、します |
| 尊敬語 | お～になる、いらっしゃる | 抬高对方 | お読みになる、ご覧になる |
| 謙譲語 | お～する、伺う、申す | 贬低自己 | お伝えする、拝見する、申し上げる |

**商务场景中，对客户 / 上司：尊敬語 + 謙譲語 双重并用**。

## 子风格

### 1. ビジネスメール（商务邮件）

**结构模板**：
```
[件名] ◯◯の件について

〇〇株式会社
△△様

いつもお世話になっております。
株式会社◯◯の□□でございます。

[本文]
……

何卒よろしくお願い申し上げます。

―――――――――――――
[署名]
―――――――――――――
```

**高频句式**：
- いつもお世話になっております。（万能开场）
- 早速ですが、〇〇の件についてご連絡いたしました。
- 誠に恐れ入りますが、……
- お手数をおかけしますが、ご確認のほどよろしくお願いいたします。
- 何卒ご査収くださいますよう、お願い申し上げます。
- ご多忙のところ恐縮ですが、……
- ご検討のほど、よろしくお願い申し上げます。

### 2. 社外敬語（正式对外）

**特征**：极致谦卑 + 全敬语 + 长定语 + 主语省略到极致。

**例**：
- 原文："資料を送ります。"
- 改写："下記の通り、資料をお送りさせていただきますので、ご査収のほどよろしくお願い申し上げます。"

### 3. 社内カジュアル（社内口语）

**特征**：去敬语 + 短句 + 助词省略。

**例**：
- 原文："この件、了解しました。"
- 改写：「この件、了解です！」 / 「りょ」 / 「把握しました」

### 4. 学会 / 論文（学术日语）

**特征**：である体（不用 です/ます） + 名词化 + 受動態。

**签名句式**：
- 「〜と考えられる」「〜と示唆される」
- 「本研究では……を検討した」
- 「……は今後の課題である」

**例**：
- 原文："我发现 A 导致 B。"
- 改写："本研究の結果、A が B に影響を与えることが示唆された。ただし、サンプルサイズの制約により、さらなる検証が今後の課題となる。"

## 婉拒 / 含蓄表达（最核心）

日语拒绝几乎不说「No」。下列是"字面 ≠ 实际"的高频表达，narikiri 输出时要用：

| 表达 | 实际意思 |
|---|---|
| 検討します / 前向きに検討します | 拒绝（99%） |
| 難しいですね | 做不到 |
| ちょっと…… | 不方便 / 做不到 |
| また今度 / 機会があれば | 大概率不会再约 |
| 善処します | 不会做任何事（官方用语） |
| 考えさせてください | 不想答应 |

## 关键助词与语尾

- **ですね / ますね** — 添加柔软度
- **かもしれません** — 降低断言
- **〜と思います** — 几乎所有个人意见都包一层
- **〜させていただきます** — 极度谦卑地"请允许我……"
- **〜ております** — "ている" 的谦语版，商务场景高频

## 改写注意

- **敬语一旦开了要坚持到底**。正文用敬语、结尾突然去敬语会显得失礼。
- **不要乱用さん / 様 / 先生**：
  - 様 — 对客户 / 正式场合
  - さん — 同事 / 一般关系
  - 先生 — 老师、医生、律师、政治家、作家
- **主语用 "私" 要谦卑化为 "わたくし"**（商务）
- **邮件不要用 emoji**（商务场景 basically 禁用）
- **年月日 / 署名 格式**要正确：「2026 年 4 月 17 日」、部门 + 姓名 + 联系方式。

## 中 / 英 / 日转换注意

- 中→日：中文的"请" / "麻烦您"在日语里要变成完整的敬语句式，不是单词替换。
- 英→日：英文的 "Please send me" 在日语里 ≠ 「送ってください」，而是「お送りいただけますでしょうか」等完整客套。
- 日→中：日语的敬语层次往中文压缩时会损失层次，可通过"您"、"敬请"、"烦请"、"拜托"等区分。
